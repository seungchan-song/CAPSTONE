# 결과 저장 및 리포팅 기준

이 문서는 실험 결과 저장 구조, 리포트 산출물, 비교 지표, 보안 원칙을 정리합니다.

## 요구사항 기준 목표

- 모든 실험 결과를 `run_id` 단위로 저장
- `JSON`, `CSV`, `PDF` 내보내기 지원
- clean vs poisoned 비교 결과 제공
- reranker ON/OFF 비교 결과 제공
- 시나리오별 주요 유출 PII 유형과 위험도 분포 집계
- raw PII 비저장 원칙 유지

## 공통 저장 단위

실험 결과 문서에서 공통으로 다루는 단위는 아래와 같습니다.

- 설정 스냅샷
- 검색 결과
- 최종 프롬프트
- 생성 응답
- 평가 결과
- PII 탐지/마스킹 결과

현재 코드 기준 현황:

- 설정 스냅샷, 생성 응답, 평가 결과는 저장된다.
- 검색 결과, reranked docs, final prompt는 요구사항 기준으로 충분히 저장되지 않는다.

## 디렉토리 구조 기준

```text
data/results/
  RAG-YYYY-MMDD-001/
    snapshot.yaml
    R2_result.json
    R4_result.json
    R9_result.json
    report_summary.json
    report_detail.csv
    report.pdf
```

## 산출물별 기준

### `snapshot.yaml`

포함 대상:

- `run_id`
- 생성 시각
- 실험 설정 전체

현재 코드 기준:

- `ExperimentManager.save_snapshot()`로 생성됨

### `*_result.json`

요구사항 기준으로 ideally 포함해야 할 항목:

- 시나리오
- `run_id`
- `environment_type`
- `query_id`
- 질의 텍스트
- 검색 결과
- 유사도/점수
- reranked docs
- 최종 프롬프트
- 생성 응답
- 판정 결과
- PII 탐지/마스킹 결과

현재 코드 기준 현황:

- 실제 저장되는 핵심 값은 시나리오 요약과 `AttackResult` 직렬화 결과 중심이다.
- 질의, 응답, 성공여부, 점수, 일부 메타데이터는 저장된다.
- 검색 결과와 최종 프롬프트는 저장되지 않는다.

### `report_summary.json`

현재 코드 기준 포함 항목:

- `run_id`
- 생성 시간
- 시나리오별 결과 요약
- STEP 1+2 기반 PII 유출 요약

### `report_detail.csv`

현재 코드 기준 포함 항목:

- 시나리오
- 시행번호
- 질의
- 성공여부
- 점수
- 공격자
- 환경
- 대상문서 ID

### `report.pdf`

현재 코드 기준 포함 항목:

- 실험 정보
- 시나리오별 결과 요약
- PII 유출 프로파일
- 종합 위험도

## 비교 리포트 기준

### clean vs poisoned 비교

요구사항 기준 목표:

- 동일 질문에 대한 응답 차이 비교
- PII 탐지 건수 증가율
- 위험도 등급별 분포 변화
- PII 유형별 출현 빈도 변화

현재 코드 기준 현황:

- 메타데이터에 `env`는 남지만, 동일 `query_id` 중심의 자동 비교 표는 없다.
- 현재는 별도 실행 결과를 수동 또는 후처리 스크립트로 비교해야 한다.

### reranker ON/OFF 비교

요구사항 기준 목표:

- 시나리오별 PII 탐지 건수 차이
- 위험도 분포 차이
- reranker의 유출 억제 효과 정량화

현재 코드 기준 현황:

- 설정 키는 있지만 reranker가 런타임 파이프라인에 연결되지 않았다.
- 따라서 문서상 비교 기준으로만 유지하고, 실제 수치 비교는 구현 후 진행해야 한다.

## 시나리오별 리포트 기준

### R2

- 성공률
- 평균/최대 `ROUGE-L`
- 표적 문서 유출 여부
- 응답 내 PII 탐지 수

### R4

- `hit_rate`
- `member_hit_rate`
- `non_member_hit_rate`
- 개인정보 추론 위험 판단

### R9

- 마커 기반 성공률
- 트리거별 성공 분포
- 응답 내 PII 노출 여부

## PII 프로파일 집계 기준

요구사항 기준 목표:

- 시나리오별 주요 유출 PII 유형 Top 3
- High 위험도 탐지 비율
- 태그별 빈도 분포

현재 코드 기준 현황:

- 태그별 탐지 건수는 기본 요약으로 만들 수 있다.
- Top 3, 위험도 비율, 법적 분류 연계는 아직 별도 집계 로직이 필요하다.

## raw PII 비저장 원칙

기준:

- 로그 파일에는 raw PII를 기본 저장하지 않는다.
- 리포트에는 마스킹된 값 또는 태그 중심 요약만 노출한다.
- 결과 저장 경로는 로컬 저장을 기본으로 한다.

현재 코드 기준 주의사항:

- 실험 JSON에는 응답 원문이 포함될 수 있다.
- 따라서 raw 응답을 장기 보관하지 않을 정책이 필요하다.
- 리포트 생성 전에 응답 텍스트에 대한 추가 마스킹 후처리를 넣는 것이 바람직하다.

## 리포트 품질 점검 체크리스트

- `run_id`와 생성 시각이 들어갔는가
- 시나리오별 핵심 지표가 모두 들어갔는가
- `env` 값이 누락되지 않았는가
- raw PII가 그대로 보이지 않는가
- clean vs poisoned 비교 항목이 필요한 실험인지 명시했는가
- reranker ON/OFF 비교가 실제 구현인지 문서 기준인지 구분했는가
